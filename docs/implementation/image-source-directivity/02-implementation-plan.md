# Image Source 指向性实施方案

## 1. 实施总原则

- 先测试，后实现
- 每一阶段测试通过后才能进入下一阶段
- 文档、日志、状态记录与代码改动同步推进
- 所有新增 Markdown 与源码统一使用 UTF-8
- 所有模块都必须定义清晰的权力范围和安全约束

## 2. 模块划分

本轮实施拆分为以下模块：

1. 文档与追溯基础设施
2. 测试基础设施与统一管理
3. CLF 解析与指向性数据校验
4. 方向坐标与角度换算
5. ISM 路径首段出射方向提取
6. ISM 声压链路并入源指向性
7. 导出与观测字段增强
8. 回归测试与验收收口

## 3. 模块实施卡

### 模块 M01: 文档与追溯基础设施

#### 权力范围

- `docs/implementation/image-source-directivity/` 下所有文档
- 日志索引、phase 索引、状态记录

#### 安全约束

- 不修改业务代码行为
- 不把新增文档混入已有乱码文档中
- 文件名全部使用 ASCII

#### 实施目标

- 建立完整文档骨架
- 固定概念方案
- 建立后续开发可追溯入口

#### 实施细则

- 建立目录结构
- 固定概念方案和实施方案
- 设计日志、phase、状态模板

#### 测试门禁

- 文档文件全部可打开
- 索引间链接路径正确
- 文档结构能支持后续 phase 填充

#### 交付物

- 概念方案文档
- 实施方案文档
- phase 索引
- 日志索引
- 状态记录

### 模块 M02: 测试基础设施与统一管理

#### 权力范围

- 项目测试目录与测试命名规则
- 与本轮改造相关的测试基线文件

#### 安全约束

- 不得把测试散落在无统一约定的位置
- 不得绕过测试直接进入功能实现
- 测试应尽量稳定、可重复执行

#### 实施目标

- 建立统一的测试组织方式
- 将 CLF、方向换算、ISM 指向性接入相关测试收敛到固定位置

#### 实施细则

- 确认项目现有 Jest 约定
- 建立本轮专用测试分组
- 明确测试样例、命名和执行顺序

#### 测试门禁

- 测试目录结构清晰
- 可单独运行本轮测试集
- 至少具备 CLF 解析基线测试

#### 交付物

- 统一测试目录
- 本轮测试样例
- 初始测试执行记录

### 模块 M03: CLF 解析与指向性数据校验

#### 权力范围

- `CLFParser`
- `DirectivityHandler`
- 与 CLF 导入相关的测试

#### 安全约束

- 不破坏现有 CLF 导入入口
- 不改变 `Omni` 模式行为
- 不在未验证编码情况下引入新的 CLF 文本样例

#### 实施目标

- 确认 CLF 文件可被正确解析
- 确认频带、角分辨率、角网格与 directivity 数据可稳定读取
- 为后续 ISM 调用建立可信数据源

#### 实施细则

- 为标准 CLF 样例建立测试
- 校验 `frequencies`、`phi`、`theta`、`angleres`
- 校验 directivity 数据维度与插值前提

#### 测试门禁

- CLF 文件读取测试通过
- `Omni` 与 `CLF` handler 基础行为测试通过

#### 交付物

- CLF 基础单测
- 解析行为记录
- 风险说明

### 模块 M04: 方向坐标与角度换算

#### 权力范围

- 方向向量到 source-local 角度转换工具
- ISM 将调用的统一角度换算入口

#### 安全约束

- 不允许在多个模块里复制不同版本的角度换算
- 不允许混用不明确的世界坐标和局部坐标约定
- 必须对边界角度做测试覆盖

#### 实施目标

- 建立唯一的方向换算入口
- 明确世界方向、局部方向、`phi/theta` 的关系

#### 实施细则

- 定义输入输出
- 以 source 局部坐标为唯一换算基准
- 处理零向量或极角边界情况

#### 测试门禁

- 基本方向样例通过
- 源旋转后角度变化测试通过
- 边界角度行为可复现

#### 交付物

- 统一角度换算函数
- 对应单测
- 角度约定说明

### 模块 M05: ISM 路径首段出射方向提取

#### 权力范围

- `ImageSourcePath`
- 路径首段方向提取相关逻辑

#### 安全约束

- 不修改镜像源树构造原则
- 不破坏现有路径有效性校验
- 直达声和反射声必须使用统一规则解释

#### 实施目标

- 从每条有效路径中稳定提取出“真实声源的首段出射方向”

#### 实施细则

- 直达声使用 `source -> receiver`
- 反射声使用 `source -> first reflection point`
- 为后续方向性查询提供标准化方向向量

#### 测试门禁

- 直达声首段方向测试通过
- 单反射路径方向测试通过
- 多阶路径首段方向稳定

#### 交付物

- 路径方向提取接口
- 单元测试
- 行为说明记录

### 模块 M06: ISM 声压链路并入源指向性

#### 权力范围

- `ImageSourcePath.arrivalPressure()`
- 相关路径声压结算逻辑

#### 安全约束

- `Omni` 模式结果必须回归兼容
- 不重复施加方向性修正
- 不把指向性错误地叠加到每次反射点

#### 实施目标

- 将路径首段出射方向对应的源指向性增益并入路径频带声压计算

#### 实施细则

- 在路径声压结算开头注入方向性修正
- 再沿用现有反射损耗和空气衰减流程
- 保证 `Omni` 与 `CLF` 走统一入口

#### 测试门禁

- `Omni` 模式回归测试通过
- `CLF` 模式方向差异测试通过
- 源旋转导致路径声压变化测试通过

#### 交付物

- 改造后的路径声压计算
- 单元测试
- 回归记录

### 模块 M07: 导出与观测字段增强

#### 权力范围

- `exportRayPathsData()`
- 调试期路径导出结构

#### 安全约束

- 只追加字段，不随意破坏现有导出结构
- 新增字段命名清晰
- 不写入难以解释的临时字段

#### 实施目标

- 使方向性结果可被导出、检查和复现实验

#### 实施细则

- 为路径导出增加方向向量、角度与方向性频带值
- 保持原元数据结构稳定

#### 测试门禁

- 导出字段完整性测试通过
- 导出值与内部计算结果一致

#### 交付物

- 增强后的路径导出结构
- 导出测试
- 调试说明

### 模块 M08: 回归测试与验收收口

#### 权力范围

- 本轮新增测试集合
- 验收文档和收口日志

#### 安全约束

- 未完成回归，不得视为收口
- 未更新日志与状态，不得关闭 phase

#### 实施目标

- 对首版功能做完整回归
- 输出明确的已完成项、限制项和后续项

#### 实施细则

- 跑完整测试集
- 汇总通过项与风险项
- 更新 phase、日志和状态记录

#### 测试门禁

- 所有目标测试通过
- 文档和日志同步完成

#### 交付物

- 回归测试结果
- 收口日志
- 状态更新

## 4. Phase 编排原则

- Phase 严格顺序推进
- 不允许跳过测试门禁
- 每个 Phase 关闭前必须更新日志和状态

## 5. 测试门禁规则

- 先补测试或确认测试缺口
- 再实现功能
- 当前模块测试不通过，禁止进入下一个模块
- `Omni` 是必须保留的回归基线

## 6. 日志记录规则

- 每次实施必须有一条索引
- 每个 phase 至少有一份详细日志
- 失败测试必须记录失败现象与修复结论

## 7. 状态更新规则

- 每次 phase 状态变更都要更新状态记录
- 模块状态和 phase 状态必须一致
- 阻塞项不能留空泛描述

## 8. 交付完成定义

完成定义包括：

- 概念方案已固定
- 实施方案已拆分为模块和 phase
- 测试已统一管理
- 源指向性已并入 ISM
- 相关测试全部通过
- 日志、phase、状态记录齐全
